# Monitoring & Logs

**Tier:** Pro | **Type:** Internal / Admin

!!! tip "Key Insight"
    Monitoring is what turns a powerful automation into a product teams can actually operate. Logs, DLQ, and worker health make it supportable at scale and keep customer trust intact.

## Overview

Includes:
- Site-level job history and run logs
- Worker health/heartbeat monitoring
- Queue depth and throughput metrics
- Failure reporting and dead letter queue management
- Admin status dashboard at /automation/status and related run-now/admin tools

## Why It Matters

Reduces the time to diagnose and fix automation issues by making every run observable instead of mysterious.
